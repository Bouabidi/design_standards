Instant Update

Instant updates are in-context action/refreshes that do not require an explicit Save action.

For processes that require the system to pause, use the standard interstial page with the spinny graphic.

Elements

  • A button or form control that triggers the instant update.
  • A visual "processing" state at the point of action.
  • Block-level visual and textual feedback confirming that an action has occured

Example

In the example below, each button is an instant update. There is no block-level Save action. The user touches/clicks a form control and the sytem update. Once the form control is activated by the user, it will switch to loading state. The loading state will persist for a prescribed minimum amount of time and persist until the system has completed the action. When the system has processed the action and the minimum amount of time has passed, the control returns to it's normal resting state.
