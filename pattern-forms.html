Forms

Forms are the face of system functionality. Users encounter forms when they create/edit content and configure features. There are a few ways to present forms in our system: in a modal, on a page and as part of a flyout. Based on where they appear, they may require different orientations: there is a horizontal alignment, where the label is to the left of the form control; and there is is a vertical aligment, where the labels are stacked on top of the form control.

Types

  • Horizontal - form label and the form element are displayed side by side, inline.
  • Vertical - form label is immediately above the form element, stacked.

Guidelines

  • Use the orientation (vertical or horizontal) shown in the wireframes, there is never a case where a form
  • There is never a case where a single form with present both types of orientation - either all vertical or all horizontal.
  • All form elements must have a corresponding label.
  • All labels must be coupled to their corresponding form element by using the - for="ID" attribute pair associating the form element ID with label.
  • All required fields must be indicated bb setting the "required" class on the form-group.
  • All required fields must be validated before form is committed to the database.
  • Do not mix form groups or grid column classes directly with input groups. Instead, nest the input group inside of the form group or grid-related element.

Structure

  • Descriptive label - with required indicator if form element is required.
  • Form element.
  • Help text - if needed (see alerts and help text for more details).
  • Validation text - instruction to user to correct failed validation.
  • Action buttons - "Save" submits form for processing and "Cancel" aborts any/all changes and does not submit changes.

Error Handling

  • Add the "required" class to all form-groups containing a required field. This adds the red asterisk to the label indicating that the input is required.
  • Validate all required form fields on "Save" and add the "has-error" class to form-groups that fail validation. This highlights the failed field.
  • On reload after failed validation, scroll the page to the first error - so it is at the top of the page.
  • Provide help text guidance to the user so they may correct the mistake.
