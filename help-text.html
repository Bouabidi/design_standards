Alerts and Help Text

Alerts and help text can help the user understand the costs of their actions, the meaning of actions they can take and a context for a deeper understanding of choices they may encounter. Adding alerts and help text should be a deliberate and thoughtful exercise where it is determined there is a real need. Product managers and user experience designers should be consulted prior to adding either where not specified in the task/requirements.

Regarding system alerts, it is assumed that the system worked as designed in all cases. When the system does not work properly or the cost of an action is high, confirmation and validation alerts are used.

Guidelines

  • Confirmation Alerts - alerts that confirm a successful system action, should only be used for in-context actions and bulk actions.
  • Delete Confirmation Alerts - only used when there is great cost and consequences of the user action.
  • Help Text - text may be added where needed to explain the meaning of labels or actions. It may be added in a number ways described below.
    • As an inline block alert.
    • As text under the form element.
    • As a NOTE and text under the form element.

Examples

Inline Alert or Help Text

Bootstrap accommodates a variety of ways to display alerts to users. Use the inline alert only in specific cases where called for in the documentation.

Well done! You successfully read this important alert message.
Heads up! This alert needs your attention, but it's not super important.
Warning! Best check yo self, you're not looking too good.
Oh snap! Change a few things up and try submitting.
<div class="alert alert-success"><strong>Well done!</strong> You successfully read this important alert message.</div>

<div class="alert alert-info"><strong>Heads up!</strong> This alert needs your attention, but it's not super important.</div>

<div class="alert alert-warning"><strong>Warning!</strong> Best check yo self, you're not looking too good.</div>

<div class="alert alert-danger"><strong>Oh snap!</strong> Change a few things up and try submitting.</div>
Modals
Informational modal from an icon .
<a id="modal-login-1" href="#modal-help" role="button" data-toggle="modal" title="help for X"> <i class="glyphicon glyphicon-question-sign"></i></a>.
Popover - This is less complicated so the help pops over in context - good for brief help .
<a href="javascript:void()" data-toggle="popover" tabindex="2" data-trigger="focus" title="Popover Help" data-content="Good for cases where a little more information or instruction may be needed."><i class="glyphicon glyphicon-question-sign"></i></a>.
Tooltip- This is a hover-activated tooltip. Tooltip on right
<a type="button" class="btn btn-default" data-toggle="tooltip" data-placement="right" title="Tooltip on right">Tooltip on right</a>
Help Block - persistent help block added below the form element.

Example of instructional text to be put here if needed.
<div class="form-horizontal">

   <div class="form-group">

      <label class="col-sm-1 control-label" for="inputPassword3">Choose</label>

      <div class="col-sm-6">

         <select class="form-control">

            <option>1</option>

         </select>

         <div class="help-block">Example of instructional text to be put here if needed.</div>

      </div>

   </div>

</div>

Help-block with "NOTE:"

Note:  Instructional text with the NOTE emphasis can be used to give it more priority. This should be used only when emphasis is truly needed.
<div class="form-horizontal">

   <div class="form-group">

      <label class="col-sm-1 control-label" for="inputPassword3">Choose</label>

      <div class="col-sm-6">

         <select class="form-control">

            <option>1</option>

         </select>

         <div class="help-block"><span class="alert-info note-alert">Note:</span>&nbsp; Instructional text with the NOTE emphasis can be used to give it more priority. This should be used only when emphasis is truly needed.</div>

      </div>

   </div>

</div>
