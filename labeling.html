Labeling & Terminology

What "things" are called is important. Labels, titles and terms are the application's voice to the end-user and therefore must resonate with them and match their mental model in order for the words to have meaning.

Labeling Guidelines

  • Text should concisely describe the element or action.
  • Avoid being verbose - no sentences.
  • Use proper case - never use lowercase words unless explicitly specified in the task documentation/requirements.
  • Action buttons text - should be consistent across features, proper case and concise.
    • Save - rarely accompanied by any other words. Avoid "Submit", "OK", "Commit", etc.
    • Cancel - always "Cancel" if accompanying the "Save" action. Never "Close", "OK" or any other word.
    • Close - only used on informational modals and never accompanied by any other action buttons.
    • Add - never accompanied with any other words because the "Add" action is always in the context the user understands.
    • Download - rarely accompanied by any other words unless explicitly required.
  • Form Labels - should concisely describe the form element.
