Buttons

Description

Use the bootstrap anchor buttons and apply the appropriate button color for its action. Grey state buttons (btn-default) can be used for all actions when their information priority is low enough and the design requires less visibility. All buttons should have role="button" and title="what action does" attributes for accessibilty compliance.

Rules & Guidelines

Button Colors

Button colors provide visual cues for users. In general, all buttons should adhere to the rules below.

Save - blue for core system funcitonality: save, edit, submit, reply, etc.

Add - green for adding content, accepting an action, etc.

Delete - red for negative or delete actions.

Cancel - white or light grey for cancel and in cases where actions need to be visually minimized.

HTML

Copy <a href="#" role="button" class="btn btn-primary" title="save">Save</a>

<a href="#" role="button" class="btn btn-success" title="add">Add</a>

<a href="#" role="button" class="btn btn-danger" title="delete">Delete</a>

<a href="#" role="button" class="btn btn-default" title="cancel">Cancel</a>



Single-action Button Order

For forms and modals, buttons are left aligned starting with the most dominant action with Cancel always the right most button.

example 1: Save Cancel

example 2: Delete Cancel

HTML

Copy <a href="#" role="button" class="btn btn-primary">Save</a> <a href="#" role="button" class="btn btn-default">Cancel</a>

<a href="#" role="button" class="btn btn-danger">Delete</a> <a href="#" role="button" class="btn btn-default">Cancel</a>



Multi-action Buttons

For cases where there is a dominant action and one or more subordinate actions.

  • Secondary Action
  • Tertiary Action

HTML

Copy <div class="btn-group">
<button class="btn btn-primary" type="button">Primary</button>
<button class="btn btn-primary dropdown-toggle" aria-expanded="false" data-toggle="dropdown" type="button"> <span class="caret"></span> <span class="sr-only">Toggle Dropdown</span> </button>
<ul class="dropdown-menu" role="menu">
<li> <a href="#">Secondary Action</a> </li>
<li> <a href="#">Tertiary Action</a> </li>
</ul>
</div>

CONSIDERATIONS

For cases where it's a multi-action button and the dominant action isn't available, do not show it as disabled but rather move the next dominant action to the primary position.

proper:
  • Tertiary Action
improper:
  • Secondary Action
  • Tertiary Action




Icon Buttons

Icon buttons can be used for a specific use case: when there are CRUD actions for list controls like those found in the user profile or HLLanding controls. Make sure to add the title="what it does" attribute to make them accessibility compliant.

Disabled State

For cases where the action is not available or ready, use the disabled state to indicate the presence of the action. This is common for forms and actions that require user input. An example is a login form that requires both a UID and Password. Until both fields are "dirty", the button is disabled. Once the user has entered something in both fields, the button is enabled.

example 1: Save Cancel

example 2: Delete Cancel

HTML

Copy <a href="#" role="button" class="btn btn-primary disabled">Save</a> <a href="#" role="button" class="btn btn-default">Cancel</a><br />
<br />
<a href="#" role="button" class="btn btn-danger disabled">Delete</a> <a href="#" role="button" class="btn btn-default">Cancel</a>
