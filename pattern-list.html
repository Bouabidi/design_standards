Lists

Lists are a common design pattern, used to organize and present administration data and user generated content throughout the system.

Types of Lists

  • Data or tabular - reports and data grids.
  • Administrative - list of users, communities, ad campaigns, etc.
  • Aggregate - list of discussion posts, library entries, blogs, etc.

Guidelines

  • Always use a bootstrap grid (col-sm-X).
  • Use a table (table-responsive) only in cases where the information is tabular data.
  • Do not use the row-hover class unless explicitly specified in the task documentation/requirements.
  • Paginate list after 25 records unless explicitly specified in the task documentation/requirements.

Structure

  • List Header
    • List-level action(s).
    • Record count - "Showing X to X of X".
    • Filter and Sort options
  • Column header
  • Item rows
    • Details column.
    • Up to 4 information columns.
    • Actions column.
  • Pagination - commonly appearing after 25 items are in the list.
