Rules for Adding CSS

Care should be taken to keep our CSS in good working order. Properly formatting, specifically targeted and non-redundant entries make good CSS. Before adding any CSS to the solution, please consider the bullet points below.

  1. Search solution for duplicate styles before adding styles. If the class doesn’t exist, add it to the appropriate file in the directory.
  2. Add new CSS to the file that is most logically aligned with the work you are performing.
  3. Make sure your CSS is properly targeted, broadly defined styles will likely cause display issues for clients. For more details about targeting see formatting and targeting.
  4. CSS must reside in a CSS file and not in the code itself. These two methods are not acceptable:
    • inline styles that use the STYLE="" to for definition.
    • style blocks that use the STYLE tag for definition.
  5. If you encounter inline CSS or style blocks during the course of working your tasks, take the time to move it to a CSS file. This may require adding a container div and a unique class name for targeting. If you encounter this and have questions, please contact Patrick Bragg.
