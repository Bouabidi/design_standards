CRUD Buttons (create, read, updated and delete)

Consistent actions and feedback on forms and form actions are essential to maintain the application's usability. When users perform CRUD actions, they should understand what is happening and anticipate change resulting from thier action. For all each of the states below, please follow instructions and design information in the task definition for more granular instructions.

Single Action Button States

  • Create : button (create, add, etc.) that either flips a div, navigates to a form page or spawns a modal - depends on the use case.
    • Use bootstrap class="btn btn-success" for create actions.
    • Example: Add
  • Save : button should be present a disabled state (add "disabled" class to anchor) until required form fields have been populated with data. This keeps the user from submitting an incomplete form.
    • Use bootstrap class="btn btn-primary" for Save actions.
    • Example, live button: Save
    • Example, disabled button: Save
  • Edit : button (edit, change, icon, etc.) that either flips a div, navigates to the form page or spawns a modal.
    • Example: Edit
  • Delete: button (delete, remove, icon, etc.) that deletes an item from the system (a least from the user’s perspective). There are two delete interactions possible:
    • Example: Delete
    • Non-prompted, unconfirmed delete – where clicking button instantaneously removes the content. This type of delete interaction is best for “low cost” items – content that is easily replicated by the user.
      • Examples: demographics, comments, tags, announcements, etc.
    • Prompted delete confirmation – where an alert is thrown to confirm this action is what was intended. A delete confirmation should be used for “high cost” item – content that is more difficult to replicate.
      • Examples: admin configuration pages, events, volunteer opportunities, communities, libraries, blogs, etc.
  • Cancel: button is used for all process forms and modals. There are two delete interactions possible:
    • Example: Cancel

Multi-Action Buttons

Multi-action buttons are used to conserve space while presenting a range of available actions. These consist of a primary action button and a dropdown with secondary actions. The user can click the primary action or explore and select other options from the dropdown.

  • Example:
    • Secondary Action
    • Tertiary Action
  • The primary action button SHOULD NEVER be disabled. If the primary action is no longer an options, remove it and elevate a secondary action (the most logical primary action) and make it the primary action.
